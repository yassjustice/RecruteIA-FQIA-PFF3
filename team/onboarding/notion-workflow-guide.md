# 📋 Notion Workflow Guide for RecruteIA Team

> **Based on the team onboarding video (Video Project 3.mp4)**

This guide explains how to use the Notion workspace to manage your tasks and track your daily workflow.

---

## 🚨 IMPORTANT: Use Your Personal Page, NOT the General Board

**Do NOT use the general board view.** Instead:

1. Go to your **personal team page** (click your name: Otman, Brahim, Chaimae, Mohammad, etc.)
2. This filters the view to show **only tasks assigned to you**
3. Work exclusively from your personal page

---

## 📊 Understanding the Views

### Sprint Backlog
All project tasks are initially created and assigned in the **Sprint Backlog** database. This is where the manager creates and assigns tasks to team members.

### Available Views
| View | Purpose |
|------|---------|
| **Table View** | Comprehensive view of all tasks with all properties |
| **Board View** | Kanban-style view grouped by status (don't use general one!) |
| **Personal Page** | ✅ **USE THIS** - Shows only your assigned tasks |

## 🔄 Task Status Workflow

Tasks must follow this **strict status progression**:

\\\
┌─────────┐    ┌─────────────┐    ┌────────┐    ┌──────┐
│ To Do   │ →  │ In Progress │ →  │ Review │ →  │ Done │
└─────────┘    └─────────────┘    └────────┘    └──────┘
                      ↑                │
                      │                ↓
                      │          ┌─────────┐
                      └──────────│ Blocked │
                                 └─────────┘
\\\

### Status Definitions

| Status | Who Moves Here | Description |
|--------|----------------|-------------|
| **To Do** | Team Member | Pull your assigned tasks here when ready to start |
| **In Progress** | Team Member | Move here **immediately** when you start working |
| **Review** | Team Member | Move here when you finish - **DO NOT mark as Done yourself!** |
| **Done** | Manager ONLY | Manager moves approved tasks here after review |
| **Blocked** | Manager | Task has issues - read manager's comment for what to fix |

---

## ✅ Your Daily Workflow

### Step 1: Check Your Tasks
1. Open Notion
2. Navigate to **Team → [Your Name]**
3. Review tasks in your **To Do** column

### Step 2: Start Working
1. Pick a task from **To Do**
2. **Immediately** move it to **In Progress**
3. Complete the work

### Step 3: Submit for Review
1. When finished, move task to **Review**
2. ⚠️ **NEVER move tasks directly to Done**
3. Wait for manager review

### Step 4: Handle Feedback
If your task is moved to **Blocked**:
1. Read the manager's comment carefully
2. Move task back to **In Progress**
3. Make the required corrections
4. Move back to **Review** when ready

---

## 🔴 Rules to Remember

1. ❌ **Never mark your own tasks as Done** - only the manager can do this
2. ❌ **Don't use the general board** - use your personal filtered page
3. ✅ **Always move to In Progress** when you start working
4. ✅ **Read Blocked comments** carefully before making corrections
5. ✅ **One task at a time** - focus on completing tasks before starting new ones

---

## 📝 Example: Chaimae's Workflow

Chaimae has 4 assigned tasks. Her first task is about **criteria definitions and scoring** based on her experience.

1. She goes to **Team → Chaimae** (her personal page)
2. Picks the criteria definition task from **To Do**
3. Moves it to **In Progress**
4. Fills out the form based on her experience
5. Moves to **Review** when complete
6. Manager reviews:
   - ✅ If approved → Manager moves to **Done**
   - ❌ If issues → Manager moves to **Blocked** with comment
7. If Blocked, Chaimae reads comment, fixes issues, resubmits
